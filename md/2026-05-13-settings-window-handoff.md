# 2026-05-13 设置页压缩 / Ikegami 设置区交接

## 本轮目标

本轮主要不是继续改悬浮窗正文渲染，而是持续压缩设置页，尤其是：

- `设置 -> 悬浮面板显示项目 -> 悬浮窗展示模式 -> Ikegami 专属布局微调`

用户这一轮连续关注的是：

- 设置页空白过大；
- Ikegami 设置区说明文字太长；
- 滑条和下拉框的标签经常被挤没；
- `样式分享码` 区域看起来像乱码；
- 各设置卡片高度不够贴内容，展开后容易显得拥挤或被裁切。

## 本轮已完成的关键改动

### 1）设置卡片改成按实际内容高度自适应

核心文件：

- `DalamudACT/UI/SettingsWindow.cs`

已完成：

- `DrawSettingCard(...)` 不再按固定的“最大行数”强行卡住高度；
- 改成：
  - 首次使用保底高度；
  - 后续按真实内容高度记忆并展开；
- 对应新增：
  - `GetAdaptiveCardHeight(...)`
  - `RememberAdaptiveCardHeight(...)`

这项改动直接解决了一个很关键的问题：

- **`样式分享码` 那块看起来像“乱码”，实际上更像是卡片高度不够导致的裁切/挤压。**

也就是说，源码里的分享码中文字符串本身并没有坏，主要是布局高度不够。

### 4）Ikegami 区域的控件标签改成“标题在上、控件在下”

为了解决用户截图里反复出现的：

- 只看到 `130 px / 1.00 / 居中`
- 但看不到这些数值到底是什么设置项

已新增这些辅助方法：

- `DrawLabeledSliderFloat(...)`
- `DrawLabeledSliderInt(...)`
- `BeginLabeledCombo(...)`
- `DrawLabeledCheckbox(...)`

现在 Ikegami 区域的滑条 / 下拉框 / 复选框统一为：

1. 上方显示标签；
2. 下方显示控件；
3. checkbox 右侧额外显示：
   - `已开启`
   - `已关闭`

这部分已经实际用于：

- `结构与显示`
- `尺寸与对齐`
- `透明度`
- `Footer 与字号`

所以目前这几块不应再出现“标签完全看不到”的情况。

## 本轮最核心的判断

这轮最重要的一点是：

> `样式分享码` 看起来像乱码，主要不是字符串坏了，而是设置卡片固定高度导致内容被裁切、重叠和挤压；把卡片改成按实际内容高度自适应后，这个问题就基本从布局层面解决了。

如果后面用户再反馈“哪一块像乱码”，优先先看：

1. 这块是不是被固定高度截断了；
2. 这块是不是标签被列宽吃掉了；
3. 再考虑是不是真的编码问题。
