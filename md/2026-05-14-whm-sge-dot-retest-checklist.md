# 2026-05-14 白魔 / 贤者 DoT 复测检查单

## 适用目的

这份清单用于验证当前工作区里那轮 `LocalStatsService.cs` 最小修正，重点确认两件事：

1. **白魔 `天辉` 是否已经不再出现明显虚高**
2. **贤者 `均衡注药II` 是否已经不再掉成 `0`**

当前复测重点已经不是“继续补 `PlayerDotCatalog.cs`”，而是：

- `activePlayerDots` 已有 state 是否会被后续更准确状态充分升级
- 创建 DoT state 时，是否稳定优先按 `statusId` 绑定 skill

---

## 复测前先确认的前提

### 1）一定要使用当前工作区里带修正的新 DLL

先重新 build：

```powershell
dotnet build E:\git\DalamudACT\DalamudACT.sln
```

然后确认使用的是：

- `E:\git\DalamudACT\output\DalamudACT.dll`

> 不要继续拿更早的旧 DLL 结论直接代表当前代码状态。

### 2）先记住这几个 ID 的换算，避免再次看错

ACT / 日志里如果看到：

- `4094`
- `5EF4`
- `5EF2`

这里按当前排查结论，应优先理解为**十六进制显示**，对应十进制分别是：

- `4094` = `16532`
- `5EF4` = `24308`
- `5EF2` = `24306`

当前 `PlayerDotCatalog.cs` 已包含这些十进制 ID，**不要再把它们当成“缺失旧 actionId”**。

## 样本要求

这轮复测**不要求同人同本**，优先看“职业 / 机制 / 事件类型”。

建议样本满足：

1. 至少有一场**持续时间够长**的 boss 战

3. 尽量不是只看小怪战
4. 目标最好是标准 hostile NPC，方便继续沿用 hostile-only DoT 对账口径

## 对账口径

ACT 侧优先继续使用：

> **hostile-only DoT**

也就是只统计同时满足以下条件的日志行：

1. 行类型是 `24|`
2. 事件种类是 `DoT`
3. 时间落在这场战斗窗口内
4. 目标 actorId 以 `4` 开头

> 不要把打到队友、自身、或其他异常目标的 `24|DoT|...` 直接算进玩家对敌 DoT。

## 白魔复测检查单

###[必看] 目标技能

- `天辉`

### 现场检查

- [ ] 白魔本场确实施放过 `天辉`
- [ ] 插件里白魔 `dotDamage-*` 不是离谱虚高
- [ ] 时间线 / 相关记录里，不再持续出现 `3W~5W` 这类异常单跳

### 结果判断

#### 可以初步判“趋于正常”的信号

满足越多越好：

- [ ] `天辉` 单跳不再长期稳定落在 `3W~5W`
- [ ] 插件总 DoT 不再明显高出 ACT hostile-only 两三倍
- [ ] 如果能看到单跳，数值更接近旧文档里提过的常规参考区间，而不是离谱暴涨

> 旧文档里的参考口径是：
>
> - 普通跳：大约 `8k ~ 9k`
> - 暴击跳：大约 `1.3万 ~ 1.4万`
>
> 这里更适合拿来判断“是否仍明显离谱”，不是拿来当绝对硬阈值。

#### 仍应判“异常”的信号

只要出现任一项，就建议继续深入排：

- [ ] 仍出现长时间 `3W~5W` 单跳
- [ ] 插件白魔 DoT 总量仍明显高于 ACT hostile-only
- [ ] 同一场里看起来像是直伤样本又被当成了后续 tick

### 若白魔仍异常，优先补记这些信息

- 本场白魔名字
- 副本 / 战斗时间窗口
- 插件白魔 `dotDamage-*`
- ACT hostile-only 白魔 DoT
- 时间线里最夸张的几跳数值
- 这些大跳是否集中出现在某次高伤技能后

## 贤者复测检查单

###[必看] 目标技能 / 状态

- 技能：`均衡注药II`
- 锚点：`注药II`
- 状态：`2615`

### 现场检查

- [ ] 贤者本场确实施放过 `均衡注药II`
- [ ] ACT hostile-only 下，贤者确实有 DoT
- [ ] 插件里贤者 `dotDamage-*` 不再掉成 `0`

### 结果判断

#### 可以初步判“趋于正常”的信号

- [ ] 只要 ACT hostile-only 有值，插件侧通常不再直接显示 `0`
- [ ] 插件总量虽然允许略保守，但至少不再整场完全漏记
- [ ] 多次施放 / 多个 tick 后，贤者 DoT 仍能持续累计

#### 仍应判“异常”的信号

- [ ] ACT hostile-only 明确有值，但插件仍是 `0`
- [ ] 插件只有极少量零散值，明显不像持续 DoT
- [ ] 同一场里白魔回正了，但贤者依然完全不记

### 若贤者仍异常，优先补记这些信息

- 本场贤者名字
- 副本 / 战斗时间窗口
- 插件贤者 `dotDamage-*`
- ACT hostile-only 贤者 DoT
- ACT 里是否能看到：
  - `均衡注药II`
  - `注药II`
  - `statusId = 2615`
- 插件是否整场为 `0`，还是中途有少量累计后又停住

## 复测结论优先级

### 情况 A：白魔正常，贤者正常

说明当前这轮最小修正大概率有效。

下一步优先级可转去：

1. `骑士`
2. `黑魔旧雷系`
3. `绝枪`

### 情况 B：白魔仍虚高，贤者正常

优先继续看：

1. 白魔 active state 后续是否仍被错误样本污染
2. `天辉` 的最近动作 / 观测样本链路是否还有串用

### 情况 C：白魔正常，贤者仍为 0

优先继续看：

1. `statusId = 2615` 是否稳定绑定到了正确 skill
2. active state 是否创建成功但后续未升级
3. `disableAverageFallback: true` 下是否因为 skill 绑定失败而直接掉 `0`

### 情况 D：白魔仍虚高，贤者仍为 0

说明当前主线问题仍然高度集中在：

- active DoT state 更新
- `statusId -> skill` 绑定顺序
- recent action 与状态链路的匹配稳定性

这时应优先继续深挖 `LocalStatsService.cs`，而不是回头先补 catalog。
