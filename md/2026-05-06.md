- 补齐悬浮统计面板的交互细节与可配置项。
- 让历史记录不仅能展示条目，还能加载对应快照回看。

## 本轮完成的工作

### 1. DPS 统计范围收口

- 当前 `DPS` 统计只会纳入：
  - `PartyList` 中的队友
  - 本地玩家自己
- 之前按名字兜底匹配的逻辑已经移除，避免把不在当前队伍中的其他对象误算进来。
- 宠物 / 召唤物仍然会通过 `OwnerId` 归属到上述可追踪对象。

相关文件：

- `DalamudACT/Stats/LocalStatsService.cs`

### 2. 悬浮窗口页签交互补齐

- 左键点击悬浮窗的 `DPS` 页签：
  - 当前已在 `DPS` 页时，会折叠成只显示页签
  - 再点一次会恢复展开
- 折叠态窗口尺寸当前固定为：
  - 宽度 `270`
  - 高度 `42`
- 右键点击悬浮窗的 `DPS` 页签：
  - 第一次打开设置窗口
  - 再右键一次关闭设置窗口

相关文件：

- `DalamudACT/UI/FloatingStatsWindow.cs`
- `DalamudACT/UI/PluginUI.cs`
- `DalamudACT/UI/StatsPanel.cs`

### 3. 设置面板补充悬浮表格配置

- 在 `悬浮面板显示项目` 中新增：
  - `玩家列最小宽度`
  - `固定列宽`
  - `表格行高`
- 这些配置会实际作用到悬浮窗里的：
  - `DPS`
  - `HPS`
  - `承伤`
  - `概览`
  - `历史记录`
- `玩家列最小宽度` 和 `表格行高` 设为 `0` 时，仍然使用自动值。

相关文件：

- `DalamudACT/Configuration/PluginConfiguration.cs`
- `DalamudACT/UI/SettingsWindow.cs`
- `DalamudACT/UI/StatsPanel.cs`

### 4. DPS 页面新增伤害量列

- `DPS` 页新增 `伤害量` 列。
- `伤害量` 列和 `职业 / 秒伤 / 死亡` 一样，可以在设置中单独隐藏。
- `总DPS` 汇总行也会显示全队总伤害。
- 占比条 tooltip 中同步补上了 `伤害量` 字段。

相关文件：

- `DalamudACT/Configuration/PluginConfiguration.cs`
- `DalamudACT/UI/SettingsWindow.cs`
- `DalamudACT/UI/StatsPanel.cs`

### 5. 伤害数值改为中文单位显示

- 当前伤害相关文本不再使用 `K / M / B`。
- 现在统一使用中文单位：
  - `万`
  - `亿`
  - `兆`
- 这套格式同时作用于：
  - `DPS` 页伤害量
  - 概览中的总伤害 / 总承伤
  - 最大伤害文本
  - 测试数据样例

相关文件：

- `DalamudACT/Stats/LocalStatsService.cs`

### 6. 历史记录支持点击回看快照

- 历史记录不再只保存区域和时长。
- 当前每条历史记录都会带一份完整 `CombatDataWrapper` 快照。
- 在悬浮窗 `历史记录` 页签中点击某条记录后：
  - 悬浮窗会加载该次记录的数据
  - `DPS / HPS / 承伤 / 概览` 都会切到这份历史快照
  - 当前选中的历史记录会高亮显示
- 当新的实时战斗开始更新时，会自动切回实时数据，并清除历史选中态。

相关文件：

- `DalamudACT/UI/StatsModels.cs`
- `DalamudACT/Stats/LocalStatsService.cs`
- `DalamudACT/UI/StatsPanel.cs`

### 7. 导入测试数据不再清空已有历史

- 点击 `导入测试数据` 时，不再先清空已有历史记录。
- 当前行为改为：
  - 保留已有历史
  - 将 3 条测试快照追加到历史中
  - 如果同一条测试记录已经存在，则按 `区域 + 时长` 覆盖更新
- 这样反复导入测试数据不会把历史清空，也不会无限重复堆积同一组测试记录。

相关文件：

- `DalamudACT/Stats/LocalStatsService.cs`

### 8. 3 条测试数据已拆成不同快照

- `导入测试数据` 现在会生成 3 组不同内容的历史快照：
  - `零式测试场`
  - `极神兵破坏作战`
  - `木人演练场`
- 这 3 组数据在以下方面都有差异：
  - 队伍构成
  - 总伤害
  - 秒伤
  - 最大伤害
  - 承伤
  - 死亡情况
- 点击历史记录切换后，可以明显看出不是同一份模板数据。

相关文件：

- `DalamudACT/Stats/LocalStatsService.cs`

### 9. 补上主 UI 回调注册

- 当前已经注册 `UiBuilder.OpenMainUi`。
- 这样 Dalamud 的插件校验不再提示“插件没有注册 main UI callback”。
- 主窗口仍然作为插件的主入口窗口。

相关文件：

- `DalamudACT/Plugin/ACT.cs`
- `DalamudACT/UI/PluginUI.cs`
