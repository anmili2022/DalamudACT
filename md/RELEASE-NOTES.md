## 一句话介绍

`DPS统计` 现在不仅能直接在游戏里看队伍输出、伤害量、承伤和历史记录，还补上了历史记录开始/结束时间、导入导出、滚动与列宽调整，以及更明确的战斗结束判定设置。

适合发在：

### 本次更新重点

#### 1. 战斗结束设置更明确

- 设置窗口拆分出独立的 `战斗结束设置`
- 支持两种结束判定：
  - `全队脱战（PartyList）即为战斗结束`
  - `全队脱战，且延迟 X 秒为战斗结束`
- 主窗口会直接显示当前判定方式

#### 2. 历史记录更完整

- 历史记录现在显示：
  - `开始时间`
  - `结束时间`
  - `时长`
- 支持导入 / 导出历史记录
- 导入导出文件固定使用插件配置目录下的 `history-records.json`

#### 3. 表格与浏览体验更好

- 历史记录页面新增滚动条
- 历史记录表格支持拖动列宽
- 历史记录单元格悬停会显示完整内容
- `概览` 页面新增滚动条

#### 4. 导入导出入口补齐

- 历史记录页顶部可直接 `导出历史记录` / `导入历史记录`
- `数据与状态` 折叠栏也补上了同样的导入导出入口
- 会直接在界面里显示历史文件路径和导入导出状态

#### 5. 插件清单与发布信息同步整理

- manifest 改为中英双语简介
- 补齐 `Tags`
- 仓库链接统一到当前 GitHub 仓库
- `repo.json` 下载地址切换到 GitHub Release 附件

## 面向玩家的说明

如果你是普通玩家，这次更新最值得注意的是：

- 历史记录现在不只是“能点进去看”，还能看到开始/结束时间
- 需要备份或迁移历史记录时，可以直接导入 / 导出
- 如果历史很多，表格现在可以滚动、调列宽，也能悬停看完整内容
- 战斗结束判定终于可以按你的习惯调整

## 更新后建议

建议更新后优先做下面几件事：

1. 打开悬浮面板，确认 `DPS` 页面显示正常
2. 去设置里确认战斗结束判定方式是否符合你的习惯
3. 如果需要备份或迁移数据，去历史记录页试一下导入 / 导出
4. 如果表格太挤，就调一下列宽，或直接拖动历史记录列宽
5. 如果暂时没有战斗，就先导入测试数据，看看历史记录和界面效果
